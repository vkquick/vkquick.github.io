    vkquivk -- Это асинхронный фреймворк для написания легкоподдерживаемых чат ботов ВКонтакте. Давайте сами в этом убедимся, создав hello world бота

    Использование¶

    hello world бот¶

    Помимо прочего, vkquick из коробки имеет терминальный инструмент bot, который представляет из себя некого менеджера Ваших проектов. Именно через него мы будем создавать бота и добавлять в него команды

    Выберите в директорию, где будет располагаться проект с ботом, перейдите в нее, и создайте бота следующей командой: 

    $ bot new hello_world
    В ответ Вы получите такое сообщение:

    Created hello_world bot. Don't forget 'cd' to bot's directory

Please, add following parameters to hello_world/config.toml under [api] section
    -- token
    -- group_id

    Для начала, перейдем в только что созданную папку hello_world: 

    cd hello_world
    Теперь нам требуется указать токен бота и ID группы в файле config.toml (о TOML можно почитать здесь). Под заголовком [api] вы увидите две переменные: token и group_id. В них нужно вставить соответствующие значения.

    Конфиг должен выглядеть примерно так: 

    [api]
token = "ab34b87e9d87a987dab34b87e9d87a987d"
group_id = 234223543
owner = "group"
version = 5.124


[longpoll]
wait = 25

    Теперь нужно добавить команду боту. Например, hello 

    $ bot com hello
    Собственно, это все! Давайте запустим его и посмотрим, что он может 
    $ bot run --reload --debug
    Переходим в лс боту и пишем hello. Вуа-ля! Вы все еще не написали ни одной строчки на Python, а уже сделали бота на нем. Круто, не правда ли?

    Но не обходиться же нам одним hello world'ом? Я предлагаю ознокмиться с тем, что сейчас находится внутри нашей папки 

    ├── LICENSE
├── README.md
├── config.toml
└── src
    ├── __init__.py
    └── hello
        ├── __init__.py
        ├── config.py
        └── main.py
    И так, по порядку

    Структура проекта¶

    LICENSE¶

    Обычный файл с MIT лицензией. Вы можете вписать туда провообладателя (то есть себя)

    README.md¶

    Классика любого проекта. Добавляйте сюда информацию о своем боте, причем как по использованию, так и для разработки

    config.toml¶

    В нем располагаются абсолютно все настройки вашего бота, начиная токеном, заканчивая фильтром дебага. О возможных паратреах этого конфига можно узнать в config_toml

    src/¶

    Здесь содержатся все команды, валидаторы, аннотационнные типы, хендлеры сигналов, да и вообще весь код

    Команды (Reactions)¶

    Каждая команда создается таким образом: 

    $ bot com <name>
    И по умолчанию состоит из двух основных файлов:

    main.py¶

    Здесь находится сам код команды, выраженный функцией. Предлагаю разобрать по порядку, из чего она состоит

    import vkquick as vq

from . import config


@vq.Cmd(names=config.NAMES)
@vq.Reaction("message_new")
def hello():
"""
Handler to command "hello"
"""
    return config.ANSWER

    Думаю, с импортами все понятно (о config.py будет чуть ниже)

    • @vq.Cmd(names=config.NAMES)

      Валидатор vkquick.validators.cmd.Cmd -- обработчик текстовой команды. Именно в нем вы указываете, какие есть имена у команды, какие префиксы, должна ли быть команда быть чувствитель к регистру символов (по умолчанию -- нет) и прочее. В данном случае мы обозначае что у команды есть одно имя --hello (содержимое находит в config.py)

    • @vq.Reaction("message_new")

      Этот декоратор указывает на то, что мы создаем реакцию на LongPoll событие message_new

    • def hello():

      Сама реакция. Async/sync. Про аргументы речь пойдет чуть ниже

    • return config.ANSWER

      Возвращает ответ туда, откуда пришло сообщение

    config.py¶

    Настройки вашей команды. Выносите сюда имена команды, префиксы и прочее. Однако, вы можете обойтись и без этого файла, но мы рекомендуем вам стараться выносить все данные о конфигурации команды именно сюда. Заметьте, что здесь лучше располагать настройки только для 1 команды, тобишь для той, которой была выделена директория. Если вы хотите задать какой-то параметр, который легко будет доступен среди всех ваших команд (например, все ваши команды начинаются с одного и того же префикса), используйте config.toml. Достать данные из него можно с помощью атрибута config у инстанса bot.Bot. Сам инстанс можно получить через vq.current.bot, либо просто назначив аргумент функции с аннотацией, например:

    # config.toml

... # [api], [lp]
[mysettings]
prefix = ["/", "!"]

    # src/foo/main.py
import vkquick as vq


@vq.Cmd(
    names="foo",
    prefs=vq.current.bot.config.mysettings.prefix
)
@vq.Reaction("message_new")
def foo(bot: vq.Bot):
    return f"My prefixes: {bot.config.mysettings.prefix}"
    # My prefixes: ['/', '!']

    О возможностях реакций¶

    Сами функции могут быть как корутинами, так и синхронными функциями, но все они поддерживают ряд дополнительных возможностей

    Аргументы¶

    Вы можете назначать аргументы в функцию, для того, чтобы использовать дополнительные значения (для получения события, объекта пользователя, отправившего сообщение, API возможностей...), при этом указывая значение аргумента в аннтоциях. Каждая аннотация должна быть унаследована от vkquick.annotypes.base.Annotype. Все возможные аннтотипы есть в vkquick.annotypes (+ vkquick.bot.Bot и vkquick.api.API).

    Например, вот так можно получить возможность использовать VK API метод:

    import vkquick as vq


@vq.Reaction("message_new")
async def foo(api: vq.API):
    users = await api.users.get(user_ids=[1])
    ...

    Вы также видели пример использования аннотипа vkquick.bot.Bot чуть выше, в описании config.py

    Другой пример -- команда, выводящая имя и фамилию пользователя, отправившего сообщение на команду /myname

    import vkquick as vq


@vq.Cmd(prefs=["/"], names=["myname"])
@vq.Reaction("message_new")
async def myname(sender: vq.Sender(), api: vq.API):
    return f"Your name: {sender:<fn> <ln>}"

    sender предластавляет из себя объект пользователя vkquick.tools.user.User с полями быстрого доступа и определнным __format__. Советуем ознакомиться с возможностями этого объекта, потому что он возвращается как в vkquick.annotypes.sender.Sender, так и в vkquick.annotypes.replied_user.RepliedUser (пользователь из переслнного сообщения), так и во многих других

    Но помимо "payload" типов (т.е. облегчающих работу с чем-либо), вы также можете указать аргументы из текстовой команды (полный список в vkquick.annotypes.command_types), например

    import vkquick as vq


@vq.Cmd(prefs=["/"], names=["set age"])
@vq.Reaction("message_new")
def age(age: vq.Integer()):
    return f"{age=}"

    Можно вызвать вот так

    Текст команды Ответ
    /set name 456 -> age=456
    /set name 678546456456 -> age=678546456456

    Вы также можете выстраивать "цепочку" аргументов, использовать списки типов, и одновременно с этим назначать payload типы. vkquick сам определит, что к чему относится

    import vkquick as vq


@vq.Cmd(prefs=["/"], names=["profile"])
@vq.Reaction("message_new")
def profile(
    page: vq.UserMention(),
    age: vq.Integer(),
    hobbies: [vq.Word()]
):
    return f"{page=} {age=} {hobbies=}"
    Текст команды Ответ
    /profile @durov 24 music, guitar prog -> page=<vq.User id=1 name="Павел Дуров"> age=24 hobbies=['music', 'guitar', 'prog']
    /profile @deknowny 15 some, foo egg , fizz -> page=<vq.User id=447532348 name="Ян Курбатов"> age=15 hobbies=['some', 'foo', 'egg', 'fizz']

    .. tip:: Разделитель у списка это r"(?:\s*,\s*|\s+)", то есть либо запятая, либо пробел(ы)

    В замен некоторых типов, можно использовать сокращения. Вместо vkquick.annotypes.command_types.Integer вы можете использовать int, вместо vkquick.annotypes.command_types.String -- str, а вместо vkquick.annotypes.command_types.List -- просто поместить тип в квадратные скобки, например, вместо такого варианта

    def foo(
    num: vq.Integer(),
    some: vq.String(),
    fizz: vq.List(vq.UserMention())
): ...

    Можно так:

    def foo(
    num: int,
    some: str,
    fizz: [vq.UserMention()]
): ...
    О скобках в аннотипах¶

    Каждый аннотип должен быть инстансом, то есть

    Верно 

    some: vq.Word()

    Не верно 

    some: vq.Word

    Исключение составляют vkquick.bot.Bot и vkquick.api.API. Тут не нужно ставить скобки

    Быстрые ответы¶

    Вместо того, чтобы ответить сообщением туда, откуда пришла команда, вместо 

    await vq.current.api
